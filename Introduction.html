Vim reference guide

    Introduction

    Vim is a text editor, powerful one at that. It has evolved from vi to vim (Vi Improved). There's also gvim (Vim with GUI).

    • Vim is not a full fledged writing software like LibreOffice Writer or MS Word. It is a text editor, handy for writing code. There are no formatting options like bold, heading, page numbers etc.
    • If you have written C program within IDE, you might have noticed that keywords are colored (syntax highlighting), automatic code indentation, etc. Vim provides all of that and more.

    Vim has programmable abilities on editing tasks. One can also execute shell commands right within Vim and incorporate their outputs. Text can be modified, sorted, regular expressions be applied for complicated search and replace. Record a series of editing commands and execute on other portions of text in same file or another file. To sum it up — for most editing tasks, no need to write a program/script, they can be managed within Vim itself.

    Now, one might wonder, what is all this need for complicated editing features? Why does a text editor require programming capablities? Why is there even a requirement to learn how to use a text editor? All one needs from editor is writing text with keyboard, use Backspace or Delete key, have some GUI buttons for opening and saving, preferrably a search and replace feature, etc.

    A simple and short answer — to reduce repititive manual task and the time taken to type a computer code. Faster editing probably makes sense. Reduce typing time? Seems absurd. But when you are in the middle of coding a large project, immersed in complicated logic, it helps to have minimal typing distractions.

    Vim-like features have been adopted across a huge variety of applications and plugins, for example:

    • less command supports vim-like navigation
    • Extensible vi layer for Emacs
    • Vimium (browser extension), qutebrowser (keyboard-driven browser with vim-like navigation), etc
    • JetBrains IdeaVim, VSCodeVim, etc
    • Huge list of Vim-like applications and plugins

    Ice Breaker

    Open a terminal and try these steps:

    • gvim ip.txt — opens a file named ip.txt for editing
      • You can also use vim if you prefer that, or if gvim is not available
    • Press i key (yes, the lowercase alphabet i, not some alien key)
    • Start typing, for example 'What a weird editor'
    • Press Esc key
    • Press : key
    • Type wq
    • Press Enter key
    • cat ip.txt — sanity check to see what you typed is saved or not

    Phew, what a complicated procedure to write a simple line of text, isn't it? This is the most challenging and confusing part for a newbie to Vim. Don't proceed any further if you haven't got a hang of this. Take help of youtube videos if you must. Master this basic procedure and you will be ready for the awesomeness of Vim.

    Material presented here is based on gvim (GUI), which has a few subtle differences compared to vim (TUI). See this stackoverflow thread for more details.

    Opening Vim

    • gvim — opens a new document when filename is not specified
    • gvim script.py — opens script.py, blank document if script.py doesn't exist
    • gvim report.log power.log area.log — opens the specified files, but only report.log is in the current buffer
      • Use :n (short for :next) to go to next buffer for editing
    • gvim -p report.log power.log area.log — open specified files in separate tabs
    • gvim -o report.log power.log area.log — open specified files as horizontal splits
    • gvim -O report.log power.log area.log — open specified files as vertical splits
    • gvim -R script.py — opens script.py in Readonly mode
      • changes can still be made and saved despite warning messages shown (for example, by using :w!)
    • gvim -M script.py — opens script.py in stricter Readonly mode
      • Vim won't allow any changes to be made unless :set modifiable is used and file can't be saved until :set write is used
    • gvim +/while script.py — open script.py and place the cursor under the first occurrence of while
    • gvim +25 script.py — open script.py and place the cursor on the 25th line
    • gvim -c 'normal =G' script.py — open script.py and execute the normal mode command =G
    • gvim -c ':%s/search/replace/g' script.py — open script.py and execute the given Command Line mode command
    • gvim -h — complete list of options

    See also stackoverflow: using Vim to interactively edit lines filtered using grep

    Modes of Operation

    Vim is best described as a modal editor. This book will mainly discuss these four modes:

    • Insert mode
    • Normal mode
    • Visual mode
    • Command Line mode

    For complete list of modes, see :h vim-modes-intro and :h mode-switching.

    Insert mode

    This is the mode where the required text is typed. Usual editing and navigating options available are Delete, Backspace and Arrow keys. Here's some smart editing shortcuts:

    • Ctrl+w delete the current word
    • Ctrl+p auto complete word based on matching words in backward direction
      • if more than one word matches, they are displayed using a drop-down list
    • Ctrl+n auto complete word based on matching words in forward direction
    • Ctrl+x Ctrl+l auto complete line
      • if more than one line matches, they are displayed using a drop-down list
    • Ctrl+t indent current line
    • Ctrl+d unindent current line

    Pressing Esc key changes the mode back to Normal mode.

    Normal mode

    This is the default mode when Vim is opened. This mode is used to run commands for editing operations like copy, delete, paste, recording, moving around file, etc. It is also referred as the Command mode.

    The following commands are commonly used to change from Normal to Insert and Command Line modes.

    Changing to Insert mode

    • i insert, change the mode to Insert mode, places the cursor to the left
    • a append, change the mode to Insert mode, places the cursor to the right
    • I place the cursor before the first non-blank character of the line (helpful for indented lines)
    • gI place the cursor before the first column of the line
    • gi place the cursor at the same position where it was left last time in the Insert mode
    • A place the cursor at the end of the line
    • o open a new line below the current line and change to Insert mode
    • O open a new line above the current line and change to Insert mode
    • s delete character under the cursor and change to Insert mode
    • S delete entire line and change to Insert mode

    Changing to Command Line mode

    • : change the mode to Command Line mode, awaits further commands
    • / change to Command Line mode for searching in the forward direction
    • ? change to Command Line mode for searching in the backward direction

    Visual mode

    Visual mode is used to edit text by selecting them first. Selection can either be done using mouse or using visual commands:

    • v start visual selection
    • V visually select current line
    • Ctrl+v visually select column(s)

    Pressing Esc key changes the mode back to the Normal mode.

    Command Line mode

    • After : or / or ? is pressed in the Normal mode, the prompt appears in the last line of the Vim window
    • This mode is used to perform file operations like save, quit, search, replace, shell commands, etc
    • Any operation is completed by pressing Enter key after which the mode changes back to the Normal mode

    Press Esc key to ignore whatever is typed and return to the Normal mode.

    Identifying current mode in gvim

    • In Insert mode, you get a blinking | cursor
      • also -- INSERT -- can be seen on the left hand side of the Command Line
    • In Normal mode, you get a blinking rectangular block cursor, something like this █
    • In Visual mode, the Command Line shows -- VISUAL -- or -- VISUAL LINE -- according to the visual command used
    • In Command Line mode, the cursor is of course on the Command Line
